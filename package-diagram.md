## Architecture Summary

**Type:** React Native Mobile Application (iOS/Android)  
**Pattern:** Layered Architecture with Context-based State Management  
**Backend:** Supabase (REST API)  
**Navigation:** Stack-based Navigation with Role-based Routing

### Layer Responsibilities

1. **Presentation Layer** - UI components, screens, navigation, and theming
2. **Business Logic Layer** - Context providers, state management, and business rules
3. **Data Access Layer** - API services, HTTP client, and backend integration
4. **External Dependencies** - Third-party libraries and services

### Key Design Patterns

- **Context API** for global state management (Auth, Language, Favorites)
- **Service Layer** pattern for API abstraction
- **Stack Navigation** with role-based routing (Customer/Staff)
- **Component-based** architecture with reusable UI components
